## What To Prepare

TCS NQT coding questions usually test whether you can convert a simple story into clean logic under time pressure. The latest pattern still favors fundamentals over very advanced DSA.

High-priority areas:

| Priority | Topics | Why It Matters |
|---|---|---|
| Very High | Arrays, strings, numbers, loops | Most coding questions are built on these |
| Very High | Sorting, hashing, frequency count | Fast improvement from brute force to optimal |
| High | Two pointers, sliding window, prefix sum | Common for O(N) solutions |
| High | Matrix, pattern printing, basic recursion | Frequently appears in easy to medium problems |
| Medium | Greedy, stack, queue, STL | Useful for Digital/Prime level |
| Medium | Programming logic MCQs | Tests language basics and output prediction |

## Preparation Rule

For every problem, write:

1. Input format
2. Output format
3. Brute force idea
4. Optimal idea
5. Edge cases
6. C++ code
7. Time and space complexity

If you cannot explain these seven points, the problem is not fully prepared yet.

## Best Study Order

1. Master C++ input/output, arrays, strings, and STL.
2. Learn pattern recognition: frequency, sorting, prefix sum, sliding window, two pointers.
3. Solve the topic-wise question bank.
4. Redo wrong problems after 48 hours.
5. Attempt timed sets without looking at notes.
