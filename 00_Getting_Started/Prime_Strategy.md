# TCS Prime Strategy — How to Score Prime Band

## What "Prime" Actually Means

TCS Prime (sometimes called "Prime Talent" or part of "TCS Digital") is the **top performance band** in TCS NQT. Candidates in this band are considered for higher-package roles, specialized teams, or priority interview slots. The exact definition and salary range has evolved across years. [VERIFY against current TCS NQT cycle communications]

Achieving Prime is not just about knowing more — it is about **speed + accuracy + coding completion**. All three must be strong simultaneously.

---

## The Three Pillars That Separate Prime from Ninja/Digital

### Pillar 1 — Accuracy Threshold

| Band | Accuracy Target (Objective Sections) |
|------|-------------------------------------|
| Ninja | ~65–70% correct (Foundation) |
| Digital | ~75–80% correct (Foundation + Advanced) |
| **Prime** | **~85%+ correct across all sections** |

- At 85%+ accuracy with negative marking of −0.33, your effective score per 10 questions = (8.5 × 1) − (1.5 × 0.33) = **8.0 net**. [VERIFY negative marking]
- A Ninja-band candidate guessing on 30% of questions loses net marks. Prime candidates **do not guess blind** — they skip if unsure.

**Rule:** If you cannot eliminate at least 2 of 4 options, **skip and move on**.

---

### Pillar 2 — Speed (Completing All Sections With Time to Review)

| Metric | Ninja Approach | Prime Approach |
|--------|---------------|---------------|
| Time per Quant question | 90–120 sec | 45–60 sec |
| Time per Reasoning question | 60–90 sec | 30–45 sec |
| Time per Verbal question | 30–45 sec | 20–30 sec |
| RC passage (4–6 Qs) | Read fully, then answer | Skim for topic, answer inference Qs from text |
| Review time left | 0–2 min | 5–10 min per section |

**How to develop this speed:**
- Do NOT start with untimed practice. Time yourself from day 1.
- After every timed practice set, identify the 2–3 question types that ate most time — drill those specifically.
- Use approximation and elimination aggressively (see shortcuts.md files per topic).

---

### Pillar 3 — Coding Completion

This is the biggest differentiator between Digital and Prime.

| Scenario | Likely Band |
|----------|------------|
| No code submitted / does not compile | Ninja territory |
| Code compiles, passes 2–3/10 test cases | Low Digital |
| Code passes 6–8/10 test cases | Strong Digital |
| **Code passes all / near-all test cases** | **Prime** |
| **Both problems fully solved** | **Prime + early finish** |

**For Prime coding, you need to:**
1. Recognize the problem pattern within 2–3 minutes (Array, HashMap, Math, DP)
2. Write clean, correct code — not brute force that TLEs on large inputs
3. Handle edge cases: empty input, single element, duplicates, negative numbers, INT overflow

| Days | Focus |

## The Prime Mindset: Accuracy First, Speed Second

Many students train for speed first and sacrifice accuracy — they end up with net negative scores on sections they thought they were fast at.

**The correct order:**
1. Master the concept (notes.md per topic)
2. Practice untimed until 90%+ correct
3. Then add a timer — gradually reduce from 120s → 90s → 60s per question
4. Mock test under exam conditions
5. Fix mistake patterns (mistakes.md per topic)

## What NOT to Do If You're Targeting Prime

| ❌ Don't Do This | ✅ Do This Instead |
|----------------|------------------|
| Solve 500 random LeetCode problems | Solve 50 targeted problems per topic correctly |
| Guess on all Coding questions | Submit even partial brute-force that passes easy test cases |
| Skip Verbal entirely | Verbal is part of Foundation — poor Verbal can drop you below Ninja |
| Practice without a timer | Every session after the first 2 days must be timed |
| Cram formulas day before | Use cheatsheet.md and revision.md; concepts should be clear |
| Panic and switch sections randomly | Follow your practiced section order from EXAM_STRATEGY.md |

## Prime vs Digital vs Ninja — At a Glance

```
PRIME   = High Accuracy (85%+) + Speed (finishes with review time) + Coding (full/near-full)
DIGITAL = Good Accuracy (75%+) + Decent Speed + Partial Coding (at least 1 problem mostly solved)
NINJA   = Clears Foundation cutoff + does not need Coding
```
