# Time Management — Section-wise Strategy

## The Core Principle

TCS NQT has **sectional timers** — time from one section does not carry to another. [VERIFY]  
This means poor time management in any section = wasted potential, even if you're fast elsewhere.

**Goal:** Finish every section 3–5 minutes early to review flagged questions.

---

## Section-wise Time Budget

### Foundation: Numerical Ability (~40 min, ~25 questions) [VERIFY counts]

| Phase | Time | Action |
|-------|------|--------|
| Quick scan | 1 min | Read all question stems (not options) — flag any that look long/complex |
| First pass | 25–28 min | Attempt all "definitely know" questions; skip anything that needs >90 sec |
| Second pass | 8–10 min | Return to skipped questions; use elimination; guess only if 2 options remain |
| Review | 2–3 min | Check flagged answers; verify unit/answer format |

**Time per question targets:**
- Easy (direct formula): 30–45 seconds
- Medium (2-step): 60–75 seconds
- Hard / DI: 90–120 seconds max — if still stuck, skip

---

### Foundation: Verbal Ability (~30 min, ~26 questions) [VERIFY counts]

| Phase | Time | Action |
|-------|------|--------|
| RC passages | 12–15 min | Read each passage once carefully, answer all its Qs before moving on |
| Grammar / Fill-in-the-Blanks | 8–10 min | Typically fast — 20–30 sec per question |
| Para Jumbles | 4–5 min | Fix first and last sentence first, then sequence middle |
| Review | 2–3 min | Verify RC answers against passage text |

**RC Strategy:** Do NOT re-read the passage for each question. Read once, answer all RC questions from that passage, move to the next passage. Fact-based Qs = scan for keyword; Inference Qs = use passage tone.

---

### Foundation: Reasoning Ability (~30 min, ~24 questions) [VERIFY counts]

| Phase | Time | Action |
|-------|------|--------|
| Series / Coding-Decoding | 6–8 min | Usually fastest — 20–30 sec per question if pattern is clear |
| Syllogism | 4–5 min | Draw Venn diagrams mentally or on scratch paper |
| Seating / Puzzle (if present) | 8–10 min | These are time sinks — budget 4–5 min max per set, skip if unclear |
| Direction / Blood Relation | 4–5 min | Draw a diagram immediately |
| Review | 2–3 min | Double-check seating/puzzle answers |

**Key rule for Reasoning:** Puzzle and Seating sets give multiple questions from one setup. If you understand the setup, all Qs are fast. If you don't understand it in 2 minutes, **skip the entire set** and return later — do NOT waste 8 minutes on a stuck setup.

---

### Advanced: Aptitude (~35 min, ~22 questions) [VERIFY]

Harder versions of Quant + Reasoning. Same time-boxing applies, but questions take longer.

| Phase | Time | Action |
|-------|------|--------|
| First pass (easy Qs) | 18–20 min | ~50–60 sec per easy question |
| Second pass (medium Qs) | 10–12 min | Use shortcut methods — do not compute in full |
| Hard Qs / Review | 5 min | Eliminate, best-guess, or skip if no idea |

---

### Advanced: Coding (~45 min, 1–2 problems) [VERIFY]

| Phase | Time | Problem 1 | Problem 2 |
|-------|------|-----------|-----------|
| Read + plan | 0–5 min | Read carefully, identify pattern | |
| Implementation | 5–30 min | Write and debug first solution | |
| Test + edge cases | 30–35 min | Try sample inputs + edge cases | Attempt Problem 2 if P1 is passing |
| Problem 2 | 35–45 min | | Write even partial/brute force |

**Critical coding time rule:**
- If Problem 1 is not solved by minute 30 → submit what you have, start Problem 2
- Always submit something — even a brute force that passes 3/10 test cases is better than no submission
- Last 3 minutes: re-read your code for off-by-one errors and null checks

## The Skip-and-Return System

Use this consistently across all sections:

```
Question appears → Read it fully (10 sec)
        │
        ▼
Can you see the approach immediately?
    YES ──→ Solve it now (within time budget)
    NO  ──→ SKIP (mark/flag it) → move on
        │
        ▼
After first pass → Return to skipped questions
        │
        ▼
Can you eliminate 2 options?
    YES ──→ Guess from remaining 2
    NO  ──→ Leave blank (negative marking) [VERIFY if applicable]
```

---

## Elimination-Before-Guessing Framework

Never guess randomly. Before marking an answer you're unsure of:

1. **Eliminate extreme options** — In Quant, an answer that is 10× larger or smaller than others is likely wrong.
2. **Eliminate option "none of these"** — Only choose it if you've actively computed and it isn't any of the others.
3. **Use units** — If the question asks for time (hours/minutes) and two options are clearly unreasonable (e.g., 0.0001 hours for a train journey), eliminate them.
4. **Use parity** — Even × Even = Even; Odd × Odd = Odd. If your computed answer is odd but options A and C are even, eliminate A and C.
5. **Plug back in** — For equations, substitute options into the original equation. Fastest option wins.

## Common Time Killers to Avoid

| Time Killer | Fix |
|------------|-----|
| Re-reading RC passage for every question | Read once, answer all Qs from memory + scan for facts |
| Full long multiplication for approximations | Round to nearest 5/10, estimate — exact is not needed |
| Getting emotionally stuck on a hard question | Hard cap: 90 sec any Quant/Reasoning Q; 2 min for DI sets |
| Writing all working in messy scratch notes | Use organized columns — label each step |
| Changing correct answers in review | Only change if you have a clear logical reason; gut instinct in review is usually wrong |
