# Data Interpretation — Tricks & Traps

## Trap 1: Wrong Base in % Change
- **Mistake:** % change = (New - Old) / New × 100
- **Fix:** Always use OLD value as denominator: (New - Old) / Old × 100

## Trap 2: Misreading Table Units
- **Mistake:** Reading 240 when the axis says "in thousands" → actual value is 2,40,000
- **Fix:** Always check unit annotations on axes and table headers

## Trap 3: Partial Sum in Average
- **Mistake:** "Average for 3 departments" — student includes 4 departments
- **Fix:** Count the exact categories specified in the question

## Trap 4: Pie Chart — % vs Degrees
- **Mistake:** "What % represents sector X?" → answering with degree value
- **Fix:** Convert degrees to % if needed: Degrees/360 × 100

## Trick 1: Eliminate Outlier Options
For "% change from Year A to Year B" — look at the magnitude of change visually before computing.
If the bar barely changed, eliminate options showing 30%+ change.

## Trick 2: Check If Numbers Are "Round"
TCS DI problems use clean numbers. If your intermediate calculation gives 247.3, recheck your reading.

## Trick 3: Ratio Questions — Simplify First
Find ratio of X to Y → calculate X and Y, then simplify.
Don't compute large numbers — find GCD first.
