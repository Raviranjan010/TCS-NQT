### 2. Grammar Rules That Appear in Sentence Completion

#### Rule 1: Subject-Verb Agreement with Collective Nouns
* **Explanation**: Collective nouns (team, committee, board) take singular verbs when acting as a unified unit, but plural verbs when members act individually.
* **TCS Trap**: *The committee **were** divided in their opinions.* (Correct, because they act individually, but students often write *was*).
* **Practice Sentences**:
  1. The jury ______ (has/have) reached a unanimous verdict. [Ans: has]
  2. The staff ______ (is/are) arguing among themselves about the schedules. [Ans: are]
  3. Our development team ______ (runs/run) daily standups. [Ans: runs]

#### Rule 2: Preposition Choice (Different from vs. Different than)
* **Explanation**: Use "different from" when comparing nouns. "Different than" is grammatically incorrect in standard formal writing.
* **TCS Trap**: *His programming style is different **than** mine.* (Incorrect. Use **from**).
* **Practice Sentences**:
  1. The interface is different ______ the prototype. [Ans: from]
  2. This implementation logic is different ______ what we discussed. [Ans: from]
  3. The final product was completely different ______ our expectations. [Ans: from]

#### Rule 3: Articles (A/An/The) — Phonetic Rules
* **Explanation**: Use "an" before vowel sounds (even if the letter is a consonant, e.g., hour, honest, heir, MP, SD). Use "a" before consonant sounds (e.g., union, university, one-time).
* **TCS Trap**: *He is **a** hourly employee.* (Incorrect. Hourly has a silent 'h', so use **an**).
* **Practice Sentences**:
  1. The client requested ______ (a/an) XML file. [Ans: an]
  2. It was ______ (a/an) unique challenge for the team. [Ans: a]
  3. She is ______ (a/an) SQL developer. [Ans: an]

#### Rule 4: Tense Consistency in Complex Sentences
* **Explanation**: If the main clause is in the past tense, subordinate clauses must maintain past-tense consistency (especially with conditionals and indirect speech).
* **TCS Trap**: *The engineer said that he **will** fix the bug tomorrow.* (Incorrect. Use **would**).
* **Practice Sentences**:
  1. She explained that the server ______ (has/had) crashed. [Ans: had]
  2. If the team had tested the API, they ______ (would/will) have spotted the bug. [Ans: would]
  3. He promised that he ______ (would/will) submit the code on Monday. [Ans: would]

#### Rule 5: Countable vs. Uncountable Nouns
* **Explanation**: Uncountable nouns (equipment, feedback, information, furniture, software) are always singular and cannot take "s" or plural verbs.
* **TCS Trap**: *The client provided several **feedbacks**.* (Incorrect. Use **feedback**).
* **Practice Sentences**:
  1. All the newly installed ______ (equipment/equipments) was tested. [Ans: equipment]
  2. We need more ______ (information/informations) to debug. [Ans: information]
  3. The development team installed new ______ (software/softwares). [Ans: software]

## 📋 PART C: Instruction Question Guide

In the actual exam, there is **1 Instruction question** with a **25-second limit**. You are shown a set of conditional commands (rules) and must apply them to a specific input to determine the output.

### ⚡ 25-Second Execution Strategy
1. **Ignore the backstory**: Skip the introductory sentences and look straight at the bulleted rules.
2. **Scan the input**: Look at the target string/word.
3. **Trace rules in order**: Check each rule sequentially. Apply the first rule that matches.
4. **Fast execution**: Do not double check; write down the result and click Next.

### 📝 Sample Instruction Questions

#### Question 1
* **Rules**:
  * If a word starts with a vowel and ends with a consonant, reverse the word.
  * If a word starts and ends with a consonant, replace all vowels with '$'.
  * If a word starts and ends with a vowel, append the length of the word at the end.
* **Input**: `APPLE`
* **Output**: `APPLE5` (Rule 3 matches since A and E are vowels; length is 5).

#### Question 2
* **Rules**:
  * If the number is even, divide by 2.
  * If the number is odd and ends in 5, subtract 5 and divide by 10.
  * If the number is odd and does not end in 5, multiply by 3 and add 1.
* **Input**: `35`
* **Output**: `3` (Rule 2 matches: odd ending in 5 $\implies (35 - 5)/10 = 3$).

#### Question 3
* **Rules**:
  * If a string contains only uppercase letters, convert it to lowercase.
  * If a string contains mixed case letters, convert all consonants to uppercase.
  * If a string contains digits, delete all vowels.
* **Input**: `Python1`
* **Output**: `Pythn1` (Rule 3 matches because it contains the digit '1').

#### Question 4
* **Rules**:
  * If code starts with '#' and ends with a digit, remove '#'.
  * If code starts with '$' and ends with a letter, replace '$' with '*'.
  * If code does not contain special symbols, reverse it.
* **Input**: `$CODEA`
* **Output**: `*CODEA` (Rule 2 matches).

#### Question 5
* **Rules**:
  * If month has 31 days, add '31' as suffix.
  * If month has 30 days, add '30' as suffix.
  * If month is February, add '28' as suffix.
* **Input**: `APRIL`
* **Output**: `APRIL30` (Rule 2 matches).

## 💡 PART E: Last-Minute Strategy

### 1. Verification Strategy (The 5-Second Check)
After typing your answer:
* **Read it aloud mentally**: Does the sentence sound natural with your word?
* **Check the Part of Speech**: If the blank is after "to", it must be a verb (infinitive). If it's before a noun, it must be an adjective.
* **Double Check Spelling**: Typo = zero marks!

### 2. Top 5 Spelling Mistakes Engineers Make in TCS Verbal
1. **Occurred** (Incorrect: *Ocured*, *Occured*) — Must have double 'c' and double 'r'.
2. **Preceding** (Incorrect: *Preceeding*) — Single 'e' in the second syllable.
3. **Obsolete** (Incorrect: *Obsolate*) — Ends in "lete".
4. **Incompatible** (Incorrect: *Incompatable*) — Ends in "ible".
5. **Simultaneously** (Incorrect: *Simultaniously*) — Ends in "eously".
