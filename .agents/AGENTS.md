# TCS NQT Preparation Content & Visual Rules

These rules represent the target standards for the TCS NQT preparation repository.

## 📐 Primary Quality Constraints
1. **No Formula Without Derivation:** Every formula must show where it comes from (even a one-line derivation).
2. **Immediate Worked Examples:** Never leave a concept or formula without a worked example immediately following it.
3. **Shortcut Demonstrations:** Never write a shortcut without demonstrating it on real numbers.
4. **Detailed PYQs:** Every PYQ-style question must include: Pattern ID $\rightarrow$ Approach $\rightarrow$ Solution $\rightarrow$ Shortcut $\rightarrow$ Variation & Trap.
5. **No Prior Knowledge Assumed:** All files must be self-contained.
6. **Break Up Theory:** Never write more than 3 consecutive sentences of theory without breaking it with an example, table, or diagram.
7. **Guided Practice:** Every practice question must contain a Hint and a detailed Solution.
8. **Coding Completeness:** Always include edge cases, complete dry runs, and time/space complexity analysis for all coding solutions.

## 🖼️ Visual Layout Standards
All concept files must include at least one of the following visuals:
- **Decision Trees:** Directing which formula or method to select.
- **Flowcharts:** Illustrating step-by-step solving sequences.
- **Comparison Tables:** Evaluating competing approaches (e.g. Brute Force vs Optimal).
- **Pattern Recognition Charts:** "If question says X $\rightarrow$ immediately use formula Y".
- **Memory Maps:** Associating related terms and concepts.
